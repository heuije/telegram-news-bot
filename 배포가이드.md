# 📡 텔레그램 봇 — Oracle Cloud 무료 서버 배포 가이드

내 노트북/데스크탑이 꺼져 있어도 24시간 봇이 작동하도록, Oracle Cloud Always Free VM에 배포합니다.
영구 무료(Always Free)이고, 가입 시 카드 인증만 필요하며 자동 과금되지 않습니다.

---

## ⛔ 0단계 — 노출된 토큰 무효화 (반드시 먼저)

채팅에 공개된 두 키를 **지금 바로** 폐기하고 새로 발급하세요.

### Telegram 토큰 재발급
1. Telegram에서 `@BotFather` 열기
2. `/mybots` → 본인 봇 선택 → `API Token` → `Revoke current token`
3. 새 토큰 복사 (`8610127507:...` 형태)

### Anthropic API 키 재발급
1. https://console.anthropic.com/settings/keys 접속
2. 기존 키 옆 `...` → **Disable** 또는 **Delete**
3. `Create Key` → 새 키 발급 → 복사 (`sk-ant-api03-...`)
4. **이 키는 한 번만 보여집니다.** 어딘가에 임시 메모해두세요.

### 본인 chat_id 확인
1. 새 봇과 텔레그램에서 아무 메시지 전송 (예: `hi`)
2. 브라우저에서 열기: `https://api.telegram.org/bot<새토큰>/getUpdates`
3. 응답 JSON의 `"chat":{"id": 6167826407, ...}` 숫자가 chat_id

## 1단계 — Oracle Cloud 계정 가입

1. https://www.oracle.com/cloud/free/ 접속 → **Start for free**
2. 이메일 인증 → **Home Region 선택**: `Korea Central (Seoul)` 또는 `Japan East (Tokyo)`
   - ⚠️ Home Region은 가입 후 **변경 불가**. 한국 사용자는 Seoul 권장.
3. 개인정보 입력 (이름, 주소)
4. **결제 카드 등록** (인증용, 자동 결제 X)
   - Always Free 자원만 쓰는 한 영구 무료
   - 한국 신용/체크카드 OK (해외결제 가능 카드)
5. SMS 인증 → 가입 완료 (5~10분 소요)

## 2단계 — Always Free VM 생성

### 가) 콘솔 접속
1. https://cloud.oracle.com/ 로그인
2. 좌측 햄버거 메뉴 → **Compute** → **Instances**
3. **Create Instance** 클릭

### 나) 인스턴스 설정
| 항목 | 설정값 |
|------|--------|
| Name | `newsbot` (아무거나 OK) |
| Compartment | 기본값 |
| **Image** | **Canonical Ubuntu 22.04** 또는 **24.04** (`Change image`로 선택) |
| **Shape** | **VM.Standard.E2.1.Micro** (AMD, Always Free 표시) |
| Networking | 기본 VCN 자동 생성 (`Create new virtual cloud network`) |
| Public IPv4 | **Assign a public IPv4 address** 체크 |
| **SSH Keys** | **Generate a key pair for me** 선택 후 **Save Private Key** 클릭 → `ssh-key-XXXX.key` 다운로드 |

> 💡 **ARM shape (VM.Standard.A1.Flex)**도 Always Free에 포함되고 더 강력하지만, 가입 직후엔 용량 부족으로 거절되는 경우가 많습니다. 일단 AMD E2.1.Micro로 시작하세요.

3. **Create** 클릭 → 1~2분 대기 → 상태 `RUNNING`
4. 인스턴스 페이지에서 **Public IP Address** 메모 (예: `132.226.XX.XX`)

## 4단계 — SSH 접속 (Windows에서)

### PowerShell 사용 (Windows 10/11 기본 탑재)
다운로드한 SSH 키를 안전한 곳에 옮기고 권한 조정:

```powershell
# 다운로드 폴더에서 안전한 곳으로 이동
mkdir $HOME\.ssh -Force
Move-Item $HOME\Downloads\ssh-key-*.key $HOME\.ssh\oracle.key

# Windows에서 SSH 키 권한 잠그기 (이거 안 하면 SSH 거부됨)
icacls $HOME\.ssh\oracle.key /inheritance:r
icacls $HOME\.ssh\oracle.key /grant:r "$($env:USERNAME):(R)"
```

접속:
```powershell
ssh -i $HOME\.ssh\oracle.key ubuntu@<인스턴스Public_IP>
```

처음 접속 시 `yes` 입력. 프롬프트가 `ubuntu@newsbot:~$`로 바뀌면 성공.

## 5단계 — 봇 파일 업로드

### 가) 로컬에서 .env 파일 먼저 만들기

`C:\Users\heuij\bot-deploy\.env.example`을 복사:

```powershell
cd C:\Users\heuij\bot-deploy
Copy-Item .env.example .env
notepad .env
```

`.env` 내용에 0단계에서 새로 발급받은 값 채우기:
```
TELEGRAM_TOKEN=새로_발급받은_토큰
TELEGRAM_CHAT_ID=6167826407
ANTHROPIC_API_KEY=sk-ant-api03-새로_발급받은_키
```
저장 후 닫기.

### 나) 파일 업로드 (PowerShell)

```powershell
# 두 개의 새 PowerShell 창에서 — 첫 창은 SSH 유지, 두번째 창에서 업로드
$IP = "<인스턴스Public_IP>"
$KEY = "$HOME\.ssh\oracle.key"

# 서버에 폴더 만들기 (SSH 창에서)
ssh -i $KEY ubuntu@$IP "mkdir -p ~/newsbot"

# 파일 전송 (로컬 창에서)
scp -i $KEY -r C:\Users\heuij\bot-deploy\* ubuntu@${IP}:~/newsbot/
# .env, .gitignore 같은 점 파일은 -r에 안 잡혀서 별도 전송:
scp -i $KEY C:\Users\heuij\bot-deploy\.env ubuntu@${IP}:~/newsbot/
```

## 6단계 — 설치 스크립트 실행 (SSH 창에서)

```bash
cd ~/newsbot
bash deploy/install.sh
```

출력 마지막에 `Active: active (running)` 보이면 성공.

### 동작 확인
```bash
# 실시간 로그 (Ctrl+C로 빠져나옴)
sudo journalctl -u newsbot -f

# 또는 최근 100줄
sudo journalctl -u newsbot -n 100 --no-pager
```

텔레그램으로 `/status` 보내서 응답 오면 완전 정상.

## 7단계 — 운영 명령어 요약

SSH로 접속한 뒤 (`ssh -i $HOME\.ssh\oracle.key ubuntu@<IP>`):

| 작업 | 명령어 |
|------|--------|
| 상태 보기 | `sudo systemctl status newsbot` |
| 재시작 | `sudo systemctl restart newsbot` |
| 정지 | `sudo systemctl stop newsbot` |
| 시작 | `sudo systemctl start newsbot` |
| 실시간 로그 | `sudo journalctl -u newsbot -f` |
| 코드 수정 후 재기동 | `bash ~/newsbot/deploy/install.sh restart` |
| 의존성까지 재설치 | `bash ~/newsbot/deploy/install.sh` |

### 코드 수정하기
서버에서 직접:
```bash
nano ~/newsbot/bot.py
bash ~/newsbot/deploy/install.sh restart
```
로컬에서 수정 후 다시 업로드:
```powershell
scp -i $HOME\.ssh\oracle.key C:\Users\heuij\bot-deploy\bot.py ubuntu@<IP>:~/newsbot/
ssh -i $HOME\.ssh\oracle.key ubuntu@<IP> "bash ~/newsbot/deploy/install.sh restart"
```

### .env 수정 (토큰 교체 등)
```bash
nano ~/newsbot/.env
sudo systemctl restart newsbot
```

## 트러블슈팅

### "Permission denied (publickey)" SSH 오류
→ Windows에서 키 권한 잠그는 단계(4단계)를 빠뜨림. 다시 `icacls` 명령 실행.

### `sudo journalctl -u newsbot -f`에 한글이 깨짐
→ `locale` 출력에서 UTF-8 확인. 안 되어 있으면:
```bash
sudo locale-gen ko_KR.UTF-8
sudo update-locale LANG=ko_KR.UTF-8
sudo systemctl restart newsbot
```

### 봇이 계속 재시작됨 (CrashLoopBackOff 비슷한 증상)
```bash
sudo journalctl -u newsbot -n 50 --no-pager
```
보통:
- `❌ 환경변수 누락:` → `.env` 위치/내용 확인 (`cat ~/newsbot/.env`)
- `Conflict: terminated by other getUpdates` → 로컬에서 봇이 아직 돌고 있음. 로컬 `bot.py` 종료.

### yfinance가 일부 ticker에서 빈 데이터
→ 정상. 휴장 시간이거나 일시적 데이터 누락. `/force` 명령어로 강제 확인.

### Oracle Cloud "Out of capacity" (인스턴스 생성 거부)
→ AMD E2.1.Micro 대신 다른 리전 시도하거나, 며칠 후 재시도. 시간대를 새벽으로 바꾸면 잘 잡힘.

## 비용 & 한도

- **VM (E2.1.Micro)**: 영구 무료 (Always Free), 1 OCPU + 1GB RAM
- **네트워크 송신**: 매월 10TB까지 무료 (봇 사용량은 GB 단위)
- **블록 스토리지**: 200GB까지 무료
- **카드는 인증용**: Always Free 한도 안에서 절대 자동 결제되지 않음

이 봇 한 개만 돌리면 무료 한도의 1%도 안 씁니다.
